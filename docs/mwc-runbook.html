6. WordPress → Funnel Integration

Flow

Fluent Form submit (WordPress) → PHP webhook POSTs to lead-intake edge function → lead-intake: persists lead + creates GHL contact + issues 15-min token → Returns: { funnel_url: "https://book.menswellnesscenters.com/book/entry?t=TOKEN" } → WordPress redirects user to funnel_url → BookEntry: exchanges token → seeds booking store → /book/symptom

9. Incident Runbook

Site Down

1. Check UptimeRobot: https://dashboard.uptimerobot.com/monitors/803146043 2. Check Vercel status: https://vercel.com/men-s-wellness-centers/mwclanders/deployments 3. Check last GitHub push: https://github.com/mwcforme/mwclanders/commits/main 4. If bad deploy: Vercel → Deployments → previous deploy → "Redeploy"

Leads Not Appearing in GHL

1. Check Supabase lead_captures table for recent rows (status = "failed") 2. Check lead-notify email — should arrive at eobrien@menswellnesscenters.com 3. If GHL failed: red alert email will arrive with capture_id and error 4. Manual GHL contact creation: https://app.gohighlevel.com/contacts/new 5. Check Supabase edge function logs: Dashboard → Edge Functions → lead-intake → Logs

WordPress Form Not Redirecting to Funnel

1. Check WP debug log for "FLUENT FORM - SUPABASE RESPONSE" 2. If response has ok=1 but no funnel_url → edge function needs redeployment 3. Deploy: Supabase Dashboard → Edge Functions → lead-intake → Redeploy 4. Or: npx supabase functions deploy lead-intake --project-ref yvuwzaxnyxoejwzqfbdf 5. Test: submit form, check for ?t= token in redirect URL

Booking Calendar Not Loading

1. Check GHL calendar IDs in ghlCalendars.ts 2. Check Supabase ghl_free_slots table for recent entries 3. Manually trigger sync: POST to /functions/v1/ghl-sync 4. Check GHL for calendar availability

JS Errors in Production

1. Check Sentry: https://sentry.io (project: mwc-booking-lp) 2. All /book/* routes are PHI-sanitized and blocked from Sentry by design 3. Check Vercel function logs for edge function errors
