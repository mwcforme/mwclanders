2. Data & Request Flows

A. WordPress Form → Booking Funnel

Fluent Form submit→ PHP webhook→ lead-intake (Supabase)→ GHL contact upsert→ 15-min token issued→ funnel_url returned→ WordPress redirects→ /book/entry?t=TOKEN
BookEntry→ wp-token-exchange→ Booking store seeded→ /book/symptom
Parallel: lead-intake also fires SendGrid email to eobrien@menswellnesscenters.com with full lead data + UTM attribution.

B. Landing Page Hero Form → Booking Funnel

TRTHeroForm submit→ useLeadSubmitController→ ghl-proxy (Supabase)→ GHL contact upsert→ Booking store seeded→ /book/symptom
Parallel: Meta CAPI→ meta-capi edge function→ Meta server-side event
Partial lead capture→ Fires on phone blur (before submit)

C. Calendar Slot Loading

User opens /book/schedule→ Query ghl_free_slots table→ Display available times
pg_cron every 30 min→ ghl-sync edge function→ GHL /free-slots API→ ghl_free_slots refreshed

D. Slot Monitoring

pg_cron every 30 min→ slot-monitor edge function→ Check zero-slot conditions→ SendGrid alert (if triggered)

13. Incident Runbook

Site Down

1. Check UptimeRobot: https://dashboard.uptimerobot.com/monitors/803146043 2. Check Vercel deployments: https://vercel.com/men-s-wellness-centers/mwclanders/deployments 3. If bad deploy → Vercel → Deployments → previous deploy → Redeploy 4. Check Sentry for JS crashes

Calendar Showing "Full" / Zero Slots

1. You will receive an email from slot-monitor if this persists >30 min 2. Check Admin Sync Panel: https://book.menswellnesscenters.com/admin/sync 3. Manually trigger sync via Admin → Sync tab 4. Check GHL calendar availability is set up for that location 5. Verify ghl-sync ran: Supabase Dashboard → Edge Functions → ghl-sync → Logs

WordPress Form Not Redirecting to Funnel

1. Check WP debug log for "FLUENT FORM - SUPABASE RESPONSE" 2. If response has ok=1 but no funnel_url → token insert failing 3. Check: does wp_intake_tokens table exist? (Yes, migration applied) 4. Check Supabase: Edge Functions → lead-intake → Logs for errors 5. Re-deploy if needed: npx supabase functions deploy lead-intake --project-ref lapvqhmmgeduuedleyhf

Leads Not Appearing in GHL

1. Check email — GHL failure alert fires immediately with lead details 2. Check Supabase: lead_captures table → filter crm_status = 'failed' 3. Verify GHL API key not expired: [GHL-PIT-REDACTED] 4. Check Supabase secrets: GHL_LOCATION_ID = Ghstz8eIsHWLeXek47dk

No Lead Notification Emails

1. Check inbox info subfolder at eobrien@menswellnesscenters.com 2. Check SendGrid activity log for bounce/spam issues 3. Check Supabase → lead-intake logs for SendGrid errors 4. Verify SENDGRID_API_KEY is set in Supabase secrets
